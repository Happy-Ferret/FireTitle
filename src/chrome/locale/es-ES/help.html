Introduction

FireTitle allows you to control the text that appears in the title bar of your Firefox windows.

You can specify this in two ways:

  • give each window its own name, which appears in the title. (You can also choose a pattern that does not display the window name).
  • specify a pattern to format the title bar.

Because most operating systems and window managers let you browse through your windows by title (using mechanisms such as Alt-Tab and the system tray), naming each window makes it easier to find the window you are looking for.

While the features of this extension may be fun and useful under a variety of different browsing styles, it was designed to empower the use of multiple browser windows, each of which contains multiple tabs grouped by functionality. Users who run Firefox with multiple profiles get even more value from fixed text that you can specify on a per-profile basis.

Opening the Options dialog

There are two ways to open the Options dialog. It is bound to the ';' accelerator key (Ctrl-; on Windows or Linux, Command-; on Mac). You can also to click the FireTitle icon on the toolbar. (If this icon doesn't appear, just right-click on a toolbar and and choose Customize... to open the Customize Toolbar dialog. Then drag the FireTitle icon to a toolbar and click Done).

Naming the current window

The simplest thing you can do with the FireTitle extension is give a window a name. As you type in the new name of your window in the Options dialog, the Title Preview section shows you how this affects the title of your window. By default, the name of the window appears in the title bar before whatever normally displays. If you don't include the window name in the title pattern, this string won't appear in your window title.

Specifying global settings

Setting a title pattern

The FireTitle extension replaces Firefox's default title bar text with text determined by a pattern you specify. Here is a list of the available pattern elements, and what they mean:

n window name The name of the browser window, which can be set as described in Naming Windows.
t document title The title of the page in the browser(in the window or the active tab), as determined by Firefox.
m document modifier The standard text in the title bar, such as "Mozilla Firefox Beta 1".
a age of the window how long ago the window was created (in minutes or hours or days). Useful if you know you want to find that window that you just opened (assuming your other windows had been around awhile before you opened it).
w when the window was opened This is basically the same information that age provides. I find age more useful (and less verbose), but this option is here, if you prefer it.
[text] the text string If you want to put a piece of text into the title of every window, enclose that text in square brackets. If you want to use a close bracket character ], you need to use two brackets ]].

This might be a little confusing, but don't worry: The Title Preview section will show you what it thinks you are trying to display, and will show you errors if it is confused. This functionality can be especially helpful if you run Firefox with more than one profile. Simply put a distinguishing piece of text in the pattern in each profile, and you can tell at a glance which windows are running in which profile.

You can also place the following two modifiers just after a pattern element to change its behavior:

number maximum length The preceding element's value will be limited to the first number characters. For example, t10 will display at most 10 characters from the document title. This is clearly more useful for some elements (such as title) than for others (like fixed text), but it is implemented for all of them.

To emulate the default behavior of Firefox with respect to titles, you could set the pattern to tm.

FireTitle's default pattern is ntm. In other words, FireTitle simply prepends the name of the window to the title text.

When you change the title pattern in the Options dialog and click OK, that change is propgated to all open windows.

Setting the default name for new windows

You can set the default name for new windows you open as you are browsing as follows:

  • Window has no name by default - new windows use only the title pattern
  • Use opening window's name by default - new windows use the name of the opening window in addition to the title pattern
  • Use opening window's name/title by default - new windows use the name and title of the original window in addition to the title pattern. This feature is somewhat verbose, but sometimes the context from which you opened a window is useful in determining what content the window contains.

As you change the title pattern in the Options dialog, the Title Preview section shows you what the title of your last window would look like with the given pattern. It tends to ignore what it can't understand (although any errors will appear in the error text area), and is not terribly fussy.
