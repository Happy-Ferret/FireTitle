Einführung

Mit FireTitle können sie den Titel eines Firefox Fensters beeinflussen.

FireTitle bietet Ihnen dazu zwei Möglichkeiten:

  • Jedes Fenster kann einen eigenen Namen erhalten, der im Titel benutzt wird. (Sie können allerdings auch ein Muster vorgeben, in dem der Name nicht benutzt wird).
  • Für den Titel kann ein Muster vorgegeben werden.

Die meisten Betriebssysteme und Fenstermanager bieten eine Möglichkeit, Fenster über ihrem Titel auszuwählen (z.B. mit Alt-Tab oder im Infobereich). Haben die Fenster aussagekräftige Namen, fällt die Suche nach einem Fenster viel leichter.

FireTitle kann für Sie hilfreich sein, ganz unabhängig davon, wie Sie Ihren Browser benutzen. Am meisten werden allerdings vermutlich Benutzer profitieren, die häufig mit vielen Fenstern arbeiten, in denen jeweils mehrere Tabs zu einem gemeinsamen Thema geöffnet sind. Benutzer, die mehrere Profile benutzen, können für jedes Profil eigene Einstellungen vornehmen.

Optionsdialog öffnen

Es gibt zwei Möglichkeiten, den Optionsdialog zu öffnen. Mit dem Tastaturkürzel ';' (Ctrl-; unter Windows oder Linux, Command-; auf dem Mac). Oder Sie klicken auf das FireTitle Icon in der Symbolleiste. (Falls das Icon nicht zu sehen ist, können Sie mit einem Rechtsklick in der Symbolleiste und Anpassen... den Konfigurationsdialog für die Symbolleiste öffnen. Ziehen Sie dann das FireTitle Icon auf die Symbolleiste und klicken Sie abschließend auf Fertig).

Namen des aktuellen Fensters setzen

Am einfachsten ist es, mit FireTitle einem Fenster einen Namen zu geben. Während Sie den Namen im Optionsdialog eingeben, zeigt die Titelvorschau den resultierenden Titel des Fensters an. Normalerweise steht der Name des Fensters am Anfang des Titels. Falls der Name allerdings nicht in Ihrem Titelmuster enthalten ist, wird er im Fenstertitel nicht angezeigt.

Allgemeine Einstellungen vornehmen

Das Titelmuster festlegen

FireTitle ersetzt den normalen Titel eines Firefox Fenster durch einen Text, dessen Muster Sie festgelegt haben. Folgende Musterelemente stehen zur Verfügung:

n Fenstername Der Name des Browser-Fensters, der wie in Namen des aktuellen Fensters setzen beschrieben festgelegt werden kann.
t Seitentitel Der Titel der im Browser angezeigten Seite (im Fenster bzw. aktiven Tab), so wie er von Firefox ermittelt wurde.
g document title The title of the active tab group, as determined by Firefox (might work only after being changed once).
m Seitenmodifikator Der Standardtext im Fenstertitel, z.B. "Mozilla Firefox Beta 1".
a Alter des Fensters Wie lange gibt es dieses Fenster bereits (in Minuten, Stunden oder Tage). Nützlich, wenn Sie das gerade geöffnete Fenster finden wollen (vorausgesetzt alle anderen Fenster existieren schon länger).
w Wann wurde das Fenster geöffnet Grundsätzlich die gleiche Information wie das Alter des Fensters. Ich finde das Alter nützlicher (und kürzer), aber vielleicht bevorzugen Sie diese Option.
[text] Text Wollen Sie einen immer gleichen Text in alle Fenstertitel einfügen, dann schließen Sie ihn in rechteckige Klammern ein. Um eine schließende rechteckige Klammer in den Text einzufügen ], müssen sie zwei davon hintereinander eingeben ]].

Das mag zunächst verwirrend scheinen, doch keine Sorte: Die Titelvorschau wird Ihnen stets das aktuelle Ergebnis Ihre Eingabe anzeigen sowie Fehler, falls Sie unsicher sind. Diese Funktionalität kann besonders hilfreich sein, falls Sie mehrere Profile benutzen. Benutzen Sie einfach einen eindeutigen Text im jedem Profil, und schon können Sie mit einem Blick feststellen, welches Profil Sie gerade benutzen.

Sie können auch die beiden folgenden Modifikatoren hinter jedes der oben genannten Elemente setzen, um dessen Verhalten zu verändern:

Zahl maximale Länge Die Länge des Elements wird auf Zahl Zeichen begrenzt. Zum Beispiel wird t10 höchstens die ersten 10 Zeichen des Seitentitels anzeigen. Dieser Modifikator ist für einige Elemente (z.B. Titel) eindeutig nützlicher als für andere (z.B. festen Text), steht aber für alle zur Verfügung.
! immer darstellen Manche Element resultieren manchmal in keinerlei Text. Zum Beispiel:
  • Das Element a erzeugt keinen Text, bis das Fenster mindestens eine Minute alt ist. Mit a! wird dann 0 Minuten dargestellt.

    • Aus Sicht von Firefox haben manche Seiten keinen Titel.
  • t liefert dann keinen Text, aber t! führt zur Anzeige von (Ohne Titel).
  • n! führt dann zur Anzeige KEIN NAME
  • m! führt dann zur Anzeige KEIN MODIFIKATOR.
Falls Sie es beunruhigend finden, dass dem Fenstertitel manchmal ein ganzer Block fehlt, benutzen Sie diesen Modifikator.

Benutzen Sie das Titelmuster tm, um das normale Verhalten von Firefox nachzuahmen.

Die Voreinstellung für das Titelmuster in FireTitle ist ntm. FireTitle fügt also standardmäßig den Namen des Fensters vor dem normalen Titel ein.

Wenn Sie im Optionsdialog das Titelmuster ändern und OK klicken, werden alle offenen Fenster aktualisiert.

Den Standardnamen für ein neues Fenster setzen

Beim Namen eines neuen Fenster können Sie zwischen drei Möglichkeiten wählen:

  • Kein Name - Neue Fenster benutzten lediglich das Titelmuster
  • Name des vorigen Fensters - Neue Fenster benutzen den Namen des usprünglichen Fenster in Kombination mit dem Titelmuster
  • Name und Titel des vorigen Fensters - Neue Fenster benutzen den Namen und den Titel des usprünglichen Fenster in Kombination mit dem Titelmuster. Dies führt zu längeren Titeln, doch manchmal liefern diese Informationen über den Ursprung eines Fenster hilfreiche Hinweise auf seinen Inhalt

Schon während Sie das Titelmuster im Optionsdialog eingeben, zeigt Ihnen die Titelvorschau, wie der Titel Ihres Fensters mit diesem Muster aussehen würde. Unzulässige Eingaben werden dabei so weit als möglich ignoriert (aber im Fehlerbereich angezeigt).
